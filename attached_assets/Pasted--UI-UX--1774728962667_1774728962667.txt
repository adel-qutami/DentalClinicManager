أنت مهندس برمجيات وخبير UI/UX متخصص في تطوير الأنظمة الطبية والمالية، ولديك خبرة في تحسين الأنظمة القائمة (Existing Systems) دون التأثير على استقرارها.

لدي نظام جاهز (مبني على Replit) لإدارة المرضى والزيارات، وأريد تطويره ليكون بمستوى احترافي مشابه للأنظمة الكبيرة (ERP / Medical Systems)، مع التركيز على سهولة الاستخدام (UX) وتنظيم الواجهة (UI).

المطلوب تنفيذ التحسينات التالية بشكل تدريجي وبدون كسر النظام الحالي:

أولاً: تطوير رقم الهاتف

* دعم الأرقام الدولية عبر فصل:

  * country_code (افتراضي +967 اليمن)
  * phone_number
* الحفاظ على البيانات القديمة بدون كسر النظام
* دمج الرقمين عند العرض فقط

ثانياً: تحسين تجربة الإدخال (Forms)

* تحويل جميع نماذج الإدخال (إضافة / تعديل):

  * إلى Popup / Modal احترافي
* يجب أن يحتوي الـ Popup على:

  * تصميم نظيف وواضح
  * إغلاق سهل (زر + الضغط خارج النافذة)
  * Validation فوري
  * رسائل خطأ واضحة
* دعم وضعين:

  * Add Mode (إضافة)
  * Edit Mode (تعديل)

ثالثاً: تصميم الجداول (Data Tables)

* عرض البيانات في جداول احترافية تشمل:

  * أهم الحقول (اسم المريض، الرقم، الحالة، التاريخ...)
  * ترتيب الأعمدة بشكل منطقي
* دعم:

  * البحث (Search)
  * الفرز (Sorting)
  * التصفية (Filtering) إن أمكن

رابعاً: عرض التفاصيل (Details View)

* عند الضغط على أي صف أو زر "عرض":

  * يظهر Popup أو صفحة جانبية (Drawer)
  * تحتوي على جميع التفاصيل بشكل منظم
  * تقسيم المعلومات (بيانات المريض / الزيارات / الدفعات)

خامساً: التعديل والحذف (Edit & Delete)

* التعديل:

  * يتم عبر Popup بنفس تصميم الإضافة
* الحذف:

  * يظهر Confirmation Dialog احترافي:

    * "هل أنت متأكد؟"
    * مع تحذير واضح
    * أزرار (تأكيد / إلغاء)
* عدم الحذف المباشر بدون تأكيد

سادساً: الحفاظ على استقرار النظام

* جميع التعديلات يجب أن تكون:

  * Backward Compatible
  * بدون كسر الكود الحالي
* استخدام Refactoring تدريجي

سابعاً: مخرجات مطلوبة

* كود Frontend (حسب بيئة Replit المستخدمة)
* أمثلة UI (HTML/CSS/JS أو framework)
* تحسين Backend إذا لزم
* اقتراح مكتبات جاهزة (مثل modals / tables)

ثامناً: معايير التصميم

* تصميم بسيط وواضح (Clean UI)
* سريع وسهل الاستخدام
* مناسب لموظفي العيادات
* تقليل عدد النقرات (Clicks)

الهدف:
تحويل النظام من واجهة تقليدية إلى نظام احترافي سهل الاستخدام، يعتمد على Popups وجداول منظمة، ويقدم تجربة مستخدم قريبة من الأنظمة الطبية الكبيرة.
